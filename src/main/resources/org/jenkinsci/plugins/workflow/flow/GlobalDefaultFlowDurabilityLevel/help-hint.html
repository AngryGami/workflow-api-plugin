This setting provides a trade-off between how fast pipelines run and their "durability", or ability to survive unexpected failures while running.
Facts:

  • The lowest durability modes (suggested) can be MUCH faster and write to disk FAR less often.
  • All pipelines persist their data when they complete/fail, are paused, or the master restarts gracefully.
  • Running pipelines that do NOT persist data with every step (lower durability settings) may lose data if Jenkins is not shut down gracefully.
  • Pipelines may not be able to resume or be visualized if they cannot persist data.
  • If a running pipeline loses data, you can always run the pipeline again.
  • Pipelines will generally write log data regardless of durability settings.
  • Some modes use an "atomic write" option - this helps ensure that files aren't overwritten or left partially written if something fails.
  • Atomic writes may place more stress on filesystems, so especially with networked storage it may be faster not to use them.

Note: Individual pipelines (and multibranch projects) may configure custom overrides to these defaults.
